# UML

Nos vamos a focalizar en diagramas de clase. **UML** es un lenguaje de 
modelado. Podemos diagramar en detalle todo de un sistema. Tiene **distintos niveles**.

# Arrays

Es una **estructura** que nos permite **almacenar muchos valores** bajo un mismo nombre. Los valores deben ser del **mismo tipo**. Son variables **no escalares**.

Una vez que son creados, su **tamaño no cambia**. Si se necesita un **tamaño dinámico**, podemos usar **ArrayList**, o sea una colección.

Vamos a preferir usar **colecciones en vez de arrays**, ya que tienen más ventajas.

Si los arrays son de **tipo primitivo** se inicializan automáticamente en (cero, falase o null).

Los valores de un array se pueden cambiar. Lo que **no se puede cambiar** es el **tamaño del array**.

## For y for each

La diferencia es que con el *for* podemos cometer errores al trabajar con el índice. En cambio con el *for each* no tenemos este problema, ya que no trabaja con índices.

# Colecciones

Son **elementos escalares**, nos permite tener varios elementos de distintos tipos en una variable. Y tiene **más herramientas** que los arrays **para resolver tareas**.
